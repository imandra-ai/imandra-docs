Module Imandra_thread_util__Fut

Concurrent Futures

Our notion of concurrency is organized around 'a Fut.t, which is a value that will resolve into a 'a in the future. This is (mostly) monadic and applicative, and the use of "let operators" should make it reasonably ergonomic.

The notion of 'a Fut.t is shared, but most combinators are in the functor Fut.Make which depends on an executor (to specify where the combinator code runs).
